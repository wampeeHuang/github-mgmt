# github-mgmt · GitHub 账号管家

> 审计每一寸仓库，不让别人的代码变成你的，不让你的代码裸奔。仓库归属三分法（原创项目/原创 Skill/配置基建），四条红线，五个执行步骤。

不是 `gh` 命令行工具——是 GitHub 账号管家。定期审计 wampeeHuang 的所有仓库，分类、查归属风险、标记缺失项、批量修复。

## 快速开始

```
> 帮我安装这个 skill：https://github.com/wampeeHuang/github-mgmt
```

## 设计哲学

### 审计先于修复
不知道有什么问题之前不修任何东西。全量枚举→逐项分类→生成方案→用户确认→执行→自检。跳过审计的修复 = 制造新问题。

### 四条红线不可讨论
第三方代码不入原创仓库、私有决策严格执行、每个仓库有 README + description、不做静默公开。红线不是建议——碰到直接停，不讨论，不绕路。

### 归属标注 = 信任基建
代码来源不清晰 → 外人以为是你的作品 → 信任赤字。归属标注（fork 标记、submodule 引用、README 来源列表）不是"额外工作"，是开源仓库的基础卫生。

### 三层知识模型：规则不存快照，快照不改规则
管理规则存储结构和审计逻辑。每次审计生成独立快照。操作记录追踪变更。三层非重叠——规则变了不影响历史快照，快照过期不污染规则。

## 诚实边界

- **只管理 wampeeHuang 的 GitHub 账号。** 不处理组织账号或其他用户
- **GitHub Projects 权限不足时无法查询 V2。** Token 缺少 `read:project` scope 时标注限制
- **不自动执行私有化/归档。** 所有变更需用户确认
