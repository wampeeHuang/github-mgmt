# README 写入标准

> 金标准来源：skill-html-showcase。每个公开仓库的 README 必须包含以下章节。

## 必须章节

### 1. 标题行
```
# 项目名 · 一句话定位
```
中文。项目名 + 分隔符 + 定位语。

### 2. Slogan/描述
```
> 一句话描述核心价值。面向谁、解决什么问题、怎么解决的。
```

### 3. 截图
```
![描述](screenshot.png)
```
至少一张。展示核心界面或产出物。文本类 skill 展示样本输出截图。

### 4. 快速开始
```
## 快速开始

\```
> 帮我安装这个 skill：https://github.com/用户名/仓库名
\```

或 clone/install/start 三步。
```

### 5. 工作流/使用说明
描述核心流程。多截图可以放这里。

### 6. 前置条件（有外部依赖时）
```
## 前置条件
- 依赖项
```

### 7. 技术栈
```
## 技术栈
- 技术A
- 技术B
```

### 8. 目录结构
```
## 目录结构
├── 文件
└── 目录/
```

### 9. 设计哲学
```
## 设计哲学

### 原则一标题
一段解释为什么这样选，不是别的方式。

### 原则二标题
...
```
每个原则 = 加粗标题 + 一段解释。说"为什么"，不说"是什么"。不列功能清单。

### 10. 诚实边界
```
## 诚实边界

- **做不到的事**：具体局限，不是免责声明
- **不适合的场景**：什么时候不该用这个项目
```
诚实陈述已知局限，不包装。不是"未来会支持"的理想清单。

### 11. 工作原理（可选，有复杂流程时）
```
## 工作原理
简述核心流程或架构决策。
```

### 12. 许可
```
## 许可
MIT
```

## 章节优先级

| 优先级 | 章节 | 何时可省略 |
|--------|------|-----------|
| 必须 | 标题、Slogan、截图、快速开始、许可 | 从不 |
| 必须 | 设计哲学、诚实边界 | 从不 |
| 建议 | 技术栈、目录结构 | 纯文档仓库 |
| 按需 | 前置条件、工作原理 | 无外部依赖/流程简单 |
| 按需 | 使用说明/界面 | 快速开始已覆盖 |

## 截图规范

- 截图文件放仓库根目录，命名为 `screenshot.png`
- 至少一张核心界面/产出物截图
- 文本输出类 skill：截最终产出样本
- 多界面：`screenshots/` 目录，README 中嵌入多张
